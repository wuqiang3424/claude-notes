## 如何使用

### 方式一：全局生效

将 `CLAUDE.md` 复制到 Claude 全局配置目录：

```bash
cp CLAUDE.md ~/.claude/CLAUDE.md
```

每次启动 Claude Code 时自动加载。

### 方式二：项目生效

将 `CLAUDE.md` 复制到项目工作目录：

```bash
cp CLAUDE.md /your-project/CLAUDE.md
```

进入该目录时自动加载。

## 约束概述

### 核心原则

| 原则 | 说明 |
|------|------|
| 回答完整性 | 用户一句话多个问题，必须全部回答 |
| 记忆不可信 | 不依赖 memory 系统，约束写入文件 |
| 运维安全底线 | 不破坏优先，其次才是解决问题 |
| 方案后果评估 | 给出方案前必须分析连锁反应和副作用 |

### 禁止操作

- ❌ 自己删除自己产生的文件（先问用户）
- ❌ MySQL 数据删除（绝对禁止）
- ❌ 任何破坏性操作未经确认

### 执行流程

**先说方案后执行（6步）：**

1. 展示完整命令
2. 分析连锁反应（至少3个副作用）
3. 列出影响范围
4. 说明回滚方案
5. 等待用户确认
6. 确认后执行
